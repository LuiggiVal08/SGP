# ANEXO A — Referencia de Diseño Documental

El presente tombo se estructuró siguiendo el patrón de diseño documental de un Proyecto Socio Tecnológico tipo, ilustrado por el tombo de referencia "Arisboutique" (Escalona, Romito y Angulo, 2025), desarrollado bajo la metodología RUP. A continuación se describe el **molde de diseño** que se adoptó para el tombo SGP, expresado como esqueleto de secciones y artefactos, sin replicar el contenido del caso ajeno.

## Esqueleto de diseño (molde)

```
PORTADA
  Cabecera institucional + título + autor + fecha

RESUMEN
INTRODUCCIÓN

PARTE I — DIAGNÓSTICO Y FORMULACIÓN
  ├─ Descripción del Proyecto
  ├─ Diagnóstico Situacional
  ├─ Descripción del Contexto (razón social, naturaleza, localización, historia)
  ├─ Organizaciones Vinculadas
  ├─ Problemas, Necesidades e Intereses
  │    ├─ Cuadro: Análisis de Actores Involucrados
  │    ├─ Cuadro: Escala de Medición
  │    ├─ Cuadro: Análisis Cualitativo (árbol de problema)
  │    ├─ Cuadro: Análisis Cuantitativo (puntuación)
  │    ├─ ÁRBOL DE PROBLEMAS (causas ▲ efectos)
  │    ├─ ÁRBOL DE OBJETIVOS (medios ▲ fines)
  │    └─ ÁRBOL DE OBJETIVOS CON ALTERNATIVAS
  ├─ Cuadro: Escala de Alternativas
  ├─ Cuadro: Análisis Cuantitativo de Alternativas
  ├─ Cuadro: Análisis Cualitativo de Alternativas
  ├─ Análisis Multi-Criterio (pesos)
  ├─ Diseño de la Alternativa Ganadora (bajo la metodología del proyecto)
  ├─ Matriz de Marco Lógico
  └─ Cronograma de Actividades

PARTE II — MARCO REFERENCIAL
PARTE III — MARCO METODOLÓGICO
PARTE IV — PRODUCTO FINAL
PARTE V — JUSTIFICACIÓN E IMPACTO SOCIAL
REFERENCIAS (APA 7)
```

## Criterios de diseño aplicados

- **Jerarquía visual:** cada nivel del árbol se dispone en una fila con columna lateral de etiqueta (FINES / OBJETIVO CENTRAL / MEDIOS), separada por conectores ▲, manteniendo consistencia de color entre los tres árboles.
- **Cuadros pareados:** todo análisis tiene su versión cualitativa y cuantitativa, más una escala de conversión, para respaldar la selección multicriterio.
- **Trazabilidad:** la puntuación del árbol de problemas alimenta la matriz de marco lógico y el cronograma por fases.
- **Separación de método:** el diseño de la alternativa ganadora se describe con la metodología propia del proyecto (en SGP: XP; en el tombo de referencia: RUP), sin mezclar ambas.

El tombo de referencia completo se conserva en `referencia_arisboutique_rup.md` del repositorio de trabajo, como modelo de diseño.

Fuente de la referencia de diseño: Escalona, Romito y Angulo (2025).
